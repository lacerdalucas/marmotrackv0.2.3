# constitution.md

## Princípios não negociáveis
1. O sistema deve ser mobile-first.
2. Toda mudança crítica deve gerar histórico.
3. O fluxo deve respeitar a sequência operacional real da marmoraria.
4. Nada de liberar produção sem projeto aprovado.
5. O MVP deve focar operação, não financeiro completo.
6. O banco deve ser escalável para módulos futuros.
7. O código deve ser organizado por domínio e fácil de manter.
8. Toda tela operacional deve buscar poucos cliques.
